# 1.1.3 하둡 컴포넌트들과 하둡 생태계
- 아파치 재단의 하둡 프레임워크 분류

| 구성 요소 | 설명 |
| --- | --- |
| 하둡 공통 | 하둡 환경에서 나머지 모듈들을 지원하는 기본 유틸리티들<br>- 운영 시스템 인증이나 파일 시스템과 같은 하둡 클러스터를 위한 필수 서비스<br>- 클러스터 시작 스크립트들이나 그 스크립트들을 위해 필요한 자바 파일<br>- 문서<br>- 하둡 프레임워크의 소스 코드 |
| HDFS | 데이터에 접속해 높은 처리량을 보여주는 파일 시스템 |
| YARN | 작업을 스케줄링하고 리소스를 관리하는 프레임워크 |
| 맵리듀스 | 크기가 큰 데이터 세트를 병렬 처리하기 위한 프레임워크 |
- 하둡 프로젝트 분류

| 구분 | 설명 |
| --- | --- |
| 하둡 공통 | 하둡 환경에서 나머지 모듈들을 지원하는 기본 유틸리티들 |
| 데이터 스토리지 | HDFS에 의해 제공된다. HBase(Hadoop Database)는 구조화된 데이터의 저장을 위해 많은 테이블들을 제공한다. |
| 운영 시스템 | 하둡 2에는 얀이라는 하둡의 운영 시스템이 있다. 리눅스와 같은 OS에 설치돼 작업들을 스케줄링하고 리소스를 관리한다. |
| 데이터 프로세싱 | 크기가 큰 데이터 세트의 분산 데이터 처리를 의미한다. (맵리듀스, 아파치 테즈(Tez), 스파크 컴퓨팅 엔진) |
| 관리 툴과 코디네이터 서비스 | 암바리(Ambari)는 하둡의 관리 및 모니터링 툴이고, 주키퍼는 분산 애플리케이션의 작업을 조직화하는 아파치의 구성 요소다. |

- 하둡 생태계 주요 컴포넌트

| 컴포넌트 | 설명 |
| --- | --- |
| 아브로(Avro) | 데이터를 간단한 바이너리 형태로 변형할 수 있는 프레임워크 |
| 플룸(Flume) | 스트리밍 데이터를 하둡으로 이동하는 데이터 이동 툴 |
| 에이치베이스(HBase) | HDFS를 스토리지에 사용하는 분산 컬럼 지향의 데이터베이스 |
| 에이치카탈로그(HCatalog) | HDFS에 저장한 데이터에 대한 관계형 뷰를 제공하는 서비스 |
| 하이브(Hive) | HDFS 데이터를 위한 분산 데이터 웨어하우스로, SQL 기반 쿼리를 제공한다. |
| 휴(Hue) | 사용자와 관리 인터페이스로 하여금 HDFS 파일을 검색하게 하고, Pig나 Hive 쿼리를 작동시키고 Oozie를 통한 워크플로를 스케줄링한다. |
| 카프카(Kafka) | 많은 양의 실시간 데이터를 처리하는 메시지 큐 프레임워크 |
| 마하웃(Mahout) | 맵리듀스를 이용해 개발된 머신 러닝 알고리즘 라이브러리 |
| 우지(Oozie) | 작업 스케줄링 툴 |
| 피그(Pig) | 대규모 데이터세트를 분석하기 위한 프레임워크. 데이터 파이프라인을 만들 수 있게 함. |
| 스쿱(Sqoop) | HDFS와 관계형 데이터베이스 사이의 데이터를 이동하기 위한 툴 |
| 스톰(Storm) | 실시간 프로세싱을 지원하는 객체 관계형 매핑 라이브러리 |
| 테즈(Tez) | 배치 프로세싱을 위한 데이터 처리 프레임워크로 인터랙티브한 쿼리 기능도 갖고 있다. |
| 주키퍼(ZooKeeper) | 하둡, HBase, Storm, Hive 그리고 Kafka와 같은 분산형 애플리케이션에 의해 사용되는 조정 서비스 |
