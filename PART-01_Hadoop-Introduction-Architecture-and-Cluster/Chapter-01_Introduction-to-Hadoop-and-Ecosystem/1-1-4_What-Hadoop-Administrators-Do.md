# 1.1.4 하둡 관리자들이 하는 일

| 항목 | 설명 |
| --- | --- |
| 필수 이해 | 관리자로서 모든 컴포넌트를 완전히 익혀야 하진 않지만 주키퍼, 피그, 하이브와 같은 주요 컴포넌트는 작동하는 원리에 대해 알 필요가 있다. |
| 시스템 설치와 업그레이드 | - standalone(로컬): 한 대의 머신에서 로컬 모드로 실행<br>- pseudo-distributed: 단일 노드에 하둡 환경을 설치해 분산 컴포넌트를 실행<br>- fully-distributed: 여러 노드에 걸친 실제 분산 클러스터 구성<br>- 업그레이드: 버전 호환성, 설정 이관, 롤링 업그레이드 등을 고려해 진행 |
| 구성 관리 | 손댈 수 있는 `노브(knobs)` 수가 많아 구성 매개변수가 복잡하다. |
| 개발자 지원 | 개발자나 데이터 분석가들이 하둡으로 작업할 수 있도록 계정을 제공하고 트러블 슈팅을 지원한다. |
| 성능 튜닝 | 맵리듀스 코드 작성 이상으로 피그(데이터 파이프 라이닝 툴)나 하이브(데이터 웨어하우스 관련 분석 작업을 맵리듀스로 추상화한 툴) 작업에 대한 최적화 |
| 모니터링과 트러블 슈팅 | 내장된 관리 툴(네임노드, 리소스매니저 UI), 나기오스(Nagios), Ganglia와 같은 툴로 클러스터 전체를 모니터링하고 관리할 수 있다. |
| 백업과 복구 | 모든 데이터를 저장하고 있는 하나의 DB가 없고 데이터들은 HDFS에 파일로 저장한다. 핵심 데이터를 보호하려면 스냅샷 기능을 사용할 수 있다. |

## 1.1.4.1 하둡 관리: 새로운 패러다임
- 장애 허용(fault-tolerant) 시스템
    - 대규모 데이터노드를 다루기 때문에 디스크 장애나 서버 다운은 전제로 두고 자동 복구와 재시도로 안정성을 확보한다.
## 1.1.4.2 하둡을 관리하기 위해 알아야 하는 것

| 언어 | 관련 내용 |
| --- | --- |
| Java | 맵리듀스, HDFS와 주요 하둡 코드는 자바로 작성돼 있고, 스파크는 자바로도 프로그래밍 가능하다. |
| Python | 스파크는 파이썬으로 프로그래밍할 수 있다. |
| Scala | 스파크는 스칼라로도 프로그래밍할 수 있고, 클러스터 성능 문제를 해결할 때 스칼라를 배우는 것이 도움 된다. |
| SQL | 하이브 같은 프레임워크를 프로그래밍하는 것은 맵리듀스에 대한 SQL 추상화를 사용한다. |
## 1.1.4.3 하둡 관리자 툴세트
- 클라우데라, 호튼웍스, 맵알과 같은 업체에서 공급된 배포판을 사용하는 경우가 많다. 
    - 클라우데라 매니저, 암바리와 같은 관리 툴을 사용한다.
